= Create GitHub Issue

NOTE: This documentation was generated with the assistance of AI. Please report any inaccuracies.

Draft and file a new GitHub issue that is ready to be picked up by the `/issue`, `/plan`, `/code` flow. This
skill only files an issue; it does not branch, plan, or write code.

== Purpose

`create-github-issue` turns a rough idea, bug report, or request into a well-formed GitHub issue — grounded in
the real codebase via `explore`, with a clear purpose statement asked of the user, and whatever supporting
context they have (linked URLs, attached files/documents, related issues) folded into the issue body. It
classifies the issue as bug vs. feature/enhancement from the stated purpose and matches it against this
repository's actual label set (never inventing a label that doesn't exist), drafts a title and a structured body
(Summary, Context, Acceptance criteria, References), and always shows the draft for explicit confirmation before
filing it — the same confirm-before-visible-action discipline `pr-description` and `pr-review` use. It is the
GitHub counterpart to `create-jira-ticket`.

== Outputs

* No file or code changes — this skill's only durable side effect is the filed GitHub issue itself.
* A drafted issue title and body (Summary, Context, Acceptance criteria, References) shown to the user before any
  action is taken.
* If the user confirms creation: a new GitHub issue filed via `gh issue create` (or GitHub MCP tools if `gh` is
  unavailable), with a label attached when Step 5 found one matching this repository's actual label set.
* If the user declines, or edits the draft first: no issue is filed until they explicitly confirm; the draft
  itself remains the deliverable.
* A final report with the issue number and URL, and a pointer to `/issue <number>` or `/plan <number>` as the
  next step.
